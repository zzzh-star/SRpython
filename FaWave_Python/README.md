# FaWave_Python: Four-channel Ethernet-based Force Sensing Platform

FaWave_Python 是一个从零开始重构、跨平台且具备现代化 UI 界面的力传感器上位机采集系统。该上位机完全采用模块化设计重新实现，包含以太网采集、协议解析、动态波形显示以及基于 Python 移植的 Keil C 三维力解耦算法。

## 🚀 软件功能

## 新增特性
1. **任务模式**：支持牵拉 (Traction) 与剪切 (Shear) 模式，并提供不同的安全阈值预警。
2. **3D 交互视图**：实时 3D 交互式装置模型显示，解析真正的 STL 文件 (`assets/models/device_model.stl`)。
- **以太网采集**：支持通过 TCP 或 UDP 每隔 20ms 发送自定义帧进行四通道电压数据（CH1~CH4）同步获取。
- **现代化 UI**：基于 PySide6 构建的商业仪器级仪表板，提供实时浅色 / 深色主题自适应（支持 Windows 原生深色标题栏）。
- **实时波形显示**：通过 `pyqtgraph` 提供超低延迟的多通道图表，支持拖拽、滚动缩放以及一键自动跟随。
- **三维力解耦**：内建 3×4 解耦矩阵算法，采用从 Keil C 移植到 Python 的逻辑，保证结果可复现且无需外置 C DLL。
- **本地化存储**：支持在采集过程中**随时一键开启/关闭异步记录**，防止 UI 渲染卡顿。未连接时也可预先选择存储目标以便触发等待模式。推荐实时采集使用 CSV，如使用 XLSX，系统会在停止保存后统一转换。
- **内建仿真模式**：脱离硬件也可展示开机建立基线、死区拦截、动态扰动及三维力波形等全部逻辑，便于教学与二次开发验证。

## ⚙️ 原始电压单位与关键配置说明

1. **真实设备的返回单位为 `V`**
   - 原始 FaWave 软件显示的 CH1~CH4 单位为 **V（伏特）**；
   - 当前 Python 上位机默认将 CH1~CH4 视为 **V**；
   - 解耦算法（`ForceDecoder`）的输入单位要求为 **V**；
   - 因此配置项 `input_scale_to_v = 1.0`。
   - ⚠️ **警告**：如果错误使用 `input_scale_to_v = 0.001`（或选择 `mV`），会导致输入电压缩小 1000 倍，这不仅会让三维力计算结果小 1000 倍，还会导致数据长期被“死区”（Deadzone）逻辑拦截，显示为 0。

2. **解耦算法与配置**
   - 系统目前只使用 **Python解耦运行中**。不再依赖、加载或使用外部的 C 语言后端 DLL 文件。高级设置不再存在相关切换。

3. **Fx / Fy / Fz 为 0 的常见原因排查**
   - `解耦未启用`：配置中 `enabled = false`。
   - `未初始化`：连接后第一帧还未到来，未提取初始化电压。
   - `正在开机建基线`：默认连接后有 5 秒的热身时间，需要收集稳定窗口，期间输出强制为 0。
   - `传感器确实没有受力`：环境平稳。
   - `当前力值在死区范围内`：当计算出来的力非常小，例如小于 0.5 N 或 1 N 时，软件会直接显示为 0。
   - `输入单位配置错误`：即上述将单位错设为 `mV`。
   - `传感器启动时没有保持无载静止`：导致基线无法建立。

## 💡 推荐使用流程

- **正式实验**：
  1. 采集模式选择“真实设备”。
  2. 点击建立连接后，请**保持传感器无载静止约 5 秒**。
  3. 等待右侧系统状态中的解耦状态从“基线建立中”变为**“已启用”**后，再开始加载实验。（如果偶尔闪现“死区内”属于正常范围）。
  4. 支持随时勾选“启用本地存储”。如果连接 timeout，优先检查设备是否供电。
  5. 保存数据时确认字段为 `CH1_V`~`CH4_V`。高频实时保存推荐使用 CSV。

- **无硬件演示**：
  1. 采集模式选择“仿真演示”。
  2. 连接后前 5 秒用于自动建基线，此时 Fx/Fy/Fz 将保持在 0。
  3. 约 6 秒后，内部仿真信号叠加动态扰动，仿真力曲线应开始发生变化。

- **算法调试**：
  1. 运行 `python tools/test_force_decoder_with_mock.py` 观察脱离 UI 时的状态流转是否正常。
  2. 运行 `python tools/check_unit_scale.py --file Data/...` 快速校验生成的数据字段是否合法。
  3. Python 移植后端在未通过严格的 C-Python 验证对比前，只建议用于调试评估。

## 📖 核心术语解释

用小白能懂的话解释专业概念：

- **三维力解耦**：传感器实际采集到的是 4 路电压信号，但我们希望得到 Fx、Fy、Fz 三个方向的力。解耦就是把 4 路电压变化通过数学矩阵转换成 3 个方向力的过程。
- **原始电压 CH1~CH4**：四个传感通道直接采集到的电压值，是最原始的数据。当前 FaWave 真实设备的单位为 V。
- **基线 / 零点**：传感器在无外力、静止状态下的参考电压。后续所有力的计算都以这个参考电压为零点。
- **开机建基线**：软件连接后，先让传感器保持无载静止几秒钟，自动记录当前稳定电压作为零点。这个过程完成前，Fx/Fy/Fz 会保持为 0。
- **差分电压 d1~d4**：当前滤波后的电压减去基线电压，表示每个通道相对于零点发生了多少变化。
- **解耦矩阵**：一个 3×4 的标定矩阵，用于把 d1~d4 转换为 Fz、Fx、Fy。矩阵通常来自传感器的标定实验。
- **Fz/Fx/Fy 顺序**：C 程序内部矩阵行顺序是 Fz、Fx、Fy，而界面显示通常是 Fx、Fy、Fz，因此程序中必须做正确映射，不能混淆。
- **EMA 滤波**：一种平滑算法，可以减少噪声和波动。EMA 会让输出更稳定，但响应会稍微变慢。
- **三点中值滤波**：用最近 3 个点中的中间值作为输入，可以抑制偶发尖峰干扰。
- **死区**：当计算出来的力非常小，例如小于 0.5 N 或 1 N 时，软件会直接显示为 0。这样可以避免零点附近的小噪声导致力值一直抖动。死区不是错误，而是为了让显示更稳定。
- **死区内**：表示解耦算法已经正常运行，但当前力值太小，被死区规则置零。因此 Fx/Fy/Fz 为 0 不代表算法没运行。
- **自动基线更新**：当软件判断传感器长时间处于稳定且小力状态时，会慢慢修正零点，用来抵消可能发生的长期零点漂移。
- **三维力归零**：用户手动把当前状态设为新的零点。适用于传感器安装后或实验前重新校准零点。

## 🛠️ 打包与运行

### 1. 安装依赖
```bash
python -m pip install -r requirements.txt
```

### 2. 源码启动
```bash
python main.py
```

### 3. 一键打包 (.exe)
在 Windows 下，通过 `build_exe.bat` 可以生成以 `one-folder` 形式提供的独立应用程序。打包完成后，程序结构会被放置在 `dist\FaWave\` 目录：
```bash
build_exe.bat
```

> **为何采用 one-folder 模式？**
>
> one-folder (非单文件模式) 可以带来极大的速度提升，同时使得配置的更新变得极为简便。在 `dist\FaWave` 生成后，如果需要在不重新编译的情况下调整解耦参数或网络选项，用户可直接修改位于 EXE 同级的 `config/default_config.json` 文件。

### 4. 迁移设备与图标缓存问题
若将 `dist\FaWave` 迁移到其他电脑进行使用，**请务必拷贝整个目录文件夹**而不仅仅是 `FaWave.exe` 单体文件。
如果遇到目标机器的任务栏依旧不显示更新后的蓝色图标（尽管文件浏览器中已展示新版图标），这往往是 Windows 系统自身图标缓存的限制。

**排错策略**:
1. 取消固定任务栏上的旧 FaWave 图标。
2. 完全退出所有运行中的 FaWave 进程。
3. 删除目标机器上的旧有 `dist` 目录。
4. 使用全新的 `dist\FaWave` 执行。
5. 必须时重启 `explorer.exe` (资源管理器) 或重启电脑。

### 5. 运行与检查测试清单
打包完成后，请双击 `dist\FaWave\FaWave.exe` 进行检查：
1. 主题与界面 Logo 能够正常读取展示，高级设置背景无变白；
2. “仿真演示” 可以连上且曲线滚动；
3. 左侧可随时勾选“启用本地存储”，曲线不产生阻塞卡顿；
4. 关闭软件后，产生的 `Data/` 或 `logs/` 文件夹准确生成在 EXE 所在的同一目录内。

## 三维模型配置
真实模型默认加载路径为 `assets/models/assembly_model.glb`，程序同样支持 GLB / GLTF / OBJ / STL 等文件格式进行加载。
* 推荐使用 GLB 格式以展示带颜色的彩色装配体。
* 如果使用了自定义的文件，请务必将其重命名为纯英文，不要在代码中使用中文路径。
* 若资源缺失或环境缺乏 OpenGL 兼容性，系统自动降级提供简化的 3D 占位图，且不影响传感器的数据采集。

## 三维模型预览
默认展示一张静态的模型占位预览图，点击卡片右上角的 **“查看 3D 模型”** 后，即会触发 lazy load 后台异步渲染逻辑载入真正的 `assembly_model.glb` 几何数据，极大程度保护采集过程不受资源挤占影响。

## 任务模式与安全报警
在系统顶部栏可随时无缝切换“牵拉模式”与“剪切模式”：
1. **牵拉模式**：探测 `组织滑脱`、`组织撕裂` 以及 `牵拉过载`。
2. **剪切模式**：探测 `缝合针脱出`、`穿刺力过大`、`组织撕裂` 以及 `拉线过紧`。

上述报警阈值和触发规则窗长，均可在 `config/default_config.json` 中配置。
